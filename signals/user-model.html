Signals from User ModelÔÉÅ

The lback.auth.models.User model emits several signals during critical user-related operations, such as password management and email verification. These signals offer valuable integration points for auditing user actions, triggering notifications, updating external systems, or implementing custom logic tied to user authentication and account management flows.

Signal Name

Description

Arguments (kwargs)

user_password_set

Emitted successfully after a user‚Äôs password has been hashed and set on the user model.

sender (User): The instance of the User model whose password was set.<br>``user`` (User): The User object itself.

user_password_set_failed

Emitted if an error occurs during the process of hashing or setting a user‚Äôs password.

sender (User): The instance of the User model on which the password set failed.<br>``user`` (User): The User object itself.<br>``exception`` (Exception): The exception object that caused the failure.

user_password_checked

Emitted after a user‚Äôs plain text password has been checked against the stored hashed password.

sender (User): The instance of the User model whose password was checked.<br>``user`` (User): The User object itself.<br>``success`` (bool): True if the password matched, False otherwise.

user_password_check_failed

Emitted if an error occurs during the process of checking a user‚Äôs password (e.g., due to an issue with the hashing library).

sender (User): The instance of the User model on which the password check failed.<br>``user`` (User): The User object itself.<br>``exception`` (Exception): The exception object that caused the failure.

user_email_verification_token_generated

Emitted after a new email verification token has been successfully generated and assigned to the user.

sender (User): The instance of the User model for which the token was generated.<br>``user`` (User): The User object itself.

Emitted when an attempt to verify a user‚Äôs email address fails (e.g., due to an invalid or expired token).

sender (User): The instance of the User model for which email verification failed.<br>``user`` (User): The User object itself.
