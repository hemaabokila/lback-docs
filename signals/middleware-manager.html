Signals from MiddlewareManagerÔÉÅ

The lback.core.middleware.MiddlewareManager orchestrates the execution of middleware components for both incoming requests and outgoing responses. This class emits a comprehensive set of signals throughout the middleware processing chain, providing deep insights into request/response flow, enabling custom logging, performance monitoring, and advanced debugging capabilities.

Signal Name

Description

Arguments (kwargs)

middleware_manager_initialized

Emitted immediately after the MiddlewareManager instance has been successfully initialized.

sender (MiddlewareManager): The instance of the middleware manager.

middleware_added

Emitted when a new middleware instance is successfully added to the manager‚Äôs chain.

sender (MiddlewareManager): The middleware manager instance.<br>``middleware_instance`` (Middleware): The middleware object that was added.<br>``middleware_name`` (str): The class name of the added middleware.

middleware_add_failed

Emitted when an attempt to add a middleware fails (e.g., if the object does not implement the Middleware Protocol).

sender (MiddlewareManager): The middleware manager instance.<br>``middleware_instance`` (Any): The object that failed to be added.<br>``middleware_name`` (str): The class name of the failed middleware.<br>``error_type`` (str): A string indicating the reason for failure (e.g., ‚Äúinvalid_type‚Äù).

middleware_removed

Emitted when a specific middleware instance is successfully removed from the manager.

sender (MiddlewareManager): The middleware manager instance.<br>``middleware_instance`` (Middleware): The middleware object that was removed.<br>``middleware_name`` (str): The class name of the removed middleware.

middleware_remove_failed

Emitted when an attempt to remove a specific middleware instance fails (e.g., if the instance was not found).

sender (MiddlewareManager): The middleware manager instance.<br>``middleware_instance`` (Middleware): The middleware object that failed to be removed.<br>``middleware_name`` (str): The class name of the failed middleware.<br>``error_type`` (str): A string indicating the reason for failure (e.g., ‚Äúnot_found‚Äù).

Emitted when an attempt to remove middlewares by class fails (e.g., if an invalid class type is provided).

sender (MiddlewareManager): The middleware manager instance.<br>``request`` (Any): The incoming request object.<br>``method`` (str): The HTTP method of the request.<br>``path`` (str): The URL path of the request.
